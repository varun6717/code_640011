Agentic BRD & FRD generation pipeline

A four-layer agentic system that ingests product context from across the enterprise, generates and validates business and functional requirement documents through chat, and reports on cost, quality, and adoption.

02 BRD generation layer // chat-driven authoring · validation · completion scoring
Authoring agent
BRD chat agent
brd_author.skill.md
Skill defines the BRD schema, required sections, and probing questions per section. Agent pulls context from the data layer and the code repo (brownfield), then drives the conversation with the PM until coverage thresholds are met.
  • Required sections & field-level prompts
  • Probing questions per topic
  • Cites data-layer sources inline
  • Drafts BRD.md incrementally
Validation agent
BRD validator
brd_validator.skill.md
Takes BRD.md as input. Cross-validates against the authoring skill's required topics, the data layer artifacts, and the code surface. Returns a completion score and section-level gap suggestions.
  • Section coverage check
  • Cross-reference against source artifacts
  • Per-section completion score
  • Suggestions returned to chat for one-click fill-in

03 FRD generation layer // functional translation · validation · acceptance
Authoring agent
FRD chat agent
frd_author.skill.md
Consumes accepted BRD.md. Skill defines functional decomposition rules: actor flows, system behaviors, data contracts, error states, NFRs. Chat supports inquiry and inline modification.
  • BRD → functional spec translation
  • Inquiry mode (ask questions of the FRD)
  • Modify-via-chat with diff preview
  • Drafts FRD.md
Validation agent
FRD validator
frd_validator.skill.md
Validates "Accepted" FRD.md against the authoring skill and the upstream BRD. Same pattern as Layer 02 — completion score, gap suggestions, in-chat remediation.
  • BRD-to-FRD traceability check
  • Testability / acceptance-criteria coverage
  • Per-section completion score
  • Remediation suggestions in chat
