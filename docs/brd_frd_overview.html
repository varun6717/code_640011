Agentic BRD & FRD generation pipeline

A five-layer agentic system that ingests product context from across the enterprise, generates and validates business and functional requirement documents through chat, decomposes the accepted FRD into controls-tracked Jira epics behind human gates, and reports on cost, quality, and adoption.

02 BRD generation layer // chat-driven authoring · validation · completion scoring
Authoring agent
BRD chat agent
brd_author.skill.md
Generic engine driven by brd_profile.<domain>.yaml. Grounds each section in the UI requirement frame + the section's selected sources, drafts what it can, then chats to fill only the gaps. Section by section, with revisiting.
  • Grounds on UI_INPUT frame + sources → chat fills gaps
  • Selective-reads context_set/ by section — manifest always loaded, expand on demand
  • Coarse code pass threads high-level impact into the early sections
  • At the code-impact section, delegates a code_impact subagent (deep dive)
  • Surfaces impact flags to the operator (human-mediated loop)
  • Cites sources inline · drafts BRD.md incrementally
Validation agent
BRD validator
brd_validator.skill.md
Takes BRD.md as input. Cross-validates against the authoring skill's required topics, the data layer artifacts, and the code surface. Returns a completion score and section-level gap suggestions.
  • Section coverage check
  • Cross-reference against source artifacts
  • Per-section completion score
  • Suggestions returned to chat for one-click fill-in
BRD.md // finalized after validation pass completion score · gated
BRD.md → FRD agent
03 FRD generation layer // functional translation · validation · acceptance
Authoring agent
FRD chat agent
frd_author.skill.md
Consumes accepted BRD.md. Skill defines functional decomposition rules: actor flows, system behaviors, data contracts, error states, NFRs. Chat supports inquiry and inline modification.
  • BRD → functional spec translation
  • Inquiry mode (ask questions of the FRD)
  • Modify-via-chat with diff preview
  • Drafts FRD.md
Validation agent
FRD validator
frd_validator.skill.md
Validates "Accepted" FRD.md against the authoring skill and the upstream BRD. Same pattern as Layer 02 — completion score, gap suggestions, in-chat remediation.
  • BRD-to-FRD traceability check
  • Testability / acceptance-criteria coverage
  • Per-section completion score
  • Remediation suggestions in chat

Authoring agent
Jira epic author
jira_author.skill.md
Decomposes the accepted FRD into a set of epics by functional area, mapping each to the input-template fields. Drafts a reviewable plan rather than writing to Jira directly.
  • FRD → epic clustering by functional area
  • Field mapping against the input template
  • Each epic linked to its source FRD requirements
  • Drafts jira_plan.json
Validation agent
Jira epic validator
jira_validator.skill.md
Validates jira_plan.json for bidirectional traceability and field completeness against the template. Returns a coverage score and gap list — same gated pattern as Layers 02–03.
  • Every FRD area covered by an epic
  • Every epic traces back to the FRD
  • Required + controls field completeness
  • Coverage score + gap suggestions
